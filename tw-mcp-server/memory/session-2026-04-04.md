## Project Overview

The **TW MCP** plugin (`wikilabs/tw-mcp`) adds a Model Context Protocol server to TiddlyWiki, letting AI assistants read, search, render, inspect, and edit wiki content. It runs as a Node.js TiddlyWiki command (`--mcp`).

### 3. Readonly Default

The server now starts in **readonly mode by default**. Write tools require `rw` or `readwrite`:

```
tiddlywiki . --mcp              # readonly (default)
tiddlywiki . --mcp rw           # read-write
tiddlywiki . --mcp rw listen    # read-write + HTTP server
```

Implementation: `mcp.js` line 36 sets `options = { readonly: true }`, `rw`/`readwrite` params set it to `false`.

### 4. Fixed `get_wiki_info` Settings

The `links[]` filter on `$:/core/ui/ControlPanel/Basics` returned nothing because the shadow was overridden by the `advanced-search-plus` plugin. Replaced with a direct `settingsList` array in `handlers/query.js`.

### 5. HashLine Edits

Ported from [oh-my-pi](https://github.com/can1357/oh-my-pi) (MIT license). Each text line gets a 2-character content hash. Edits reference lines by `LINE#HASH` anchors — only changed lines are sent. Hash mismatch = automatic conflict detection.

**Files:**
- `commands/hashline.js` — hash generation (Node.js crypto, NIBBLE_STR alphabet), format output, tag parsing, edit application (bottom-up), conflict detection
- Hash function: MD5 → lower 8 bits → 2-char lookup from `ZPMQVRWSNKTXJBYH` alphabet
- Not identical to oh-my-pi hashes (they use xxHash32/Bun) but self-consistent

**get_tiddler default format is now hashline:**
- `get_tiddler(detailed=true)` → fields + hashlined text (default)
- `get_tiddler(detailed=true, format="tid")` → plain text
- `get_tiddler(detailed=true, format="json")` → pure JSON

### 6. Unsafe Field Detection

Tiddlers with multiline field values, control characters, or `:` / `#` in field names can't be represented in `.tid` format. `get_tiddler` now detects these (same check as TW's `generateTiddlerFileInfo`) and outputs a **hybrid format**: pretty JSON header for fields + hashlined text body.

```
{
    "title": "asdf",
    "aaa": "multiline\nvalue"
}

1#JR:text line 1
2#MV:
3#ZR:line 3
```

Safe fields still use tid headers (more compact).

### 7. `edit_tiddler` — Text + Field Edits

Extended `edit_tiddler` to support field operations alongside text edits:

```json
{
  "title": "MyTiddler",
  "edits": [{"op": "replace_line", "pos": "3#KV", "lines": ["new text"]}],
  "set_fields": {"bbb": "some value"},
  "delete_fields": ["old-field"]
}
```

- `edits` — text line edits (optional)
- `set_fields` — add/update fields (optional)
- `delete_fields` — remove fields (optional)
- `title` is the only required parameter

### 8. Dual HTTP Server Fix

When two processes both run `--mcp listen`, the second process now checks the `.tw-mcp/connect` discovery file for a `listen: true` flag. If the primary already serves HTTP, the second process skips starting its own HTTP server — no takeover needed.

**Files changed:**
- `mcp-lib.js` — added `listen` flag to `writeDiscoveryFile`, exported `readDiscoveryFile`
- `mcp.js` — checks discovery before starting HTTP

**Status:** Implemented but not tested yet.

### 9. Server Instructions (System Prompt)

The `instructions` field in the MCP `initialize` response now includes:

- **Safety rules** — never write unless explicitly asked
- **System tiddler exclusion** — `!is[system]` first
- **Token efficiency** — pre-formatted results, present directly
- **Tool selection** — trigger words determine tool choice:

## Key Design Decisions

| Decision | Rationale |
|---|---|
| Readonly default | Safety-first. New users explore without risk. `rw` opt-in for writes. |
| Hashline as default format | AI almost always needs hashes for editing. ~5 char overhead per line is negligible. |
| JSON fallback for unsafe fields | Same check as TW filesystem. Prevents ambiguous output for multiline field values. |
| `getFieldStringBlock()` for field output | TW core function, consistent with filesystem serialization. |
| Trigger words in server instructions | LLM picks the right tool without separate training. |
| `edit_tiddler` combines text + field edits | One call is cheaper than multiple. Fields are key/value, text is line-based. |
| HashLine uses Node.js crypto (not SJCL) | Works server-side. Future: swap to SJCL for browser compatibility (devtools). |

## Lessons Learned

1. **Always use MCP tools for tiddler content** — file edits via `Edit` tool don't sync to the wiki store. Use `put_tiddler`/`edit_tiddler` for tiddlers, `Edit` only for plugin JS files (need restart anyway).
2. **Hashline read when you intend to edit** — `get_tiddler(detailed=true)` returns hashes by default, so the AI always has what it needs for `edit_tiddler`.
3. **Don't read twice** — go straight to hashline format when the user's words signal an edit intent.
4. **Tiddler rename = create new + delete old** — never delete first, create first to be safe.
5. **Omit `type` field** when it's `text/vnd.tiddlywiki` (the default) — saves tokens.
